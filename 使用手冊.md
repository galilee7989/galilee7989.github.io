## 二、最常做的事：新增一篇禱告信（推薦用網頁後台）

1. 打開瀏覽器到 https://app.pagescms.org ，用 **GitHub 帳號 galilee7989** 登入。
2. 開啟專案 `galilee7989.github.io`。
3. 左側點「**禱告信**」。
4. 捲到最下面，點「**＋ Add an entry**（新增）」。
5. 填寫：
   - **標題**：例如「2026年08月禱告信」
   - **網址代碼**：例如「2026-08」（用「年-月」，不要有空格）
   - **日期**：選當月日期
   - **PDF 檔案**：點上傳 → 按「Upload」→ 選電腦或手機裡的 PDF → 選它
6. 右上角按「**Save**」。

完成！網站約 1～2 分鐘後自動更新，新的禱告信會自動排到最前面。

> 手機也可以做同樣的步驟，用手機瀏覽器登入 app.pagescms.org 即可。

## 三、新增照片（網頁後台）

1. app.pagescms.org 登入後，左側點「**照片**」。
2. 最下面點「Add an entry」。
3. 上傳圖片、填「說明文字」。
4. 按「Save」。首頁與照片區約 1～2 分鐘後更新。

## 四、更新後多久會出現？怎麼確認？

- 按 Save 後，GitHub 會自動重新產生網站，通常 **1～2 分鐘**。
- 到 https://galilee.org.tw 重新整理（電腦按 Ctrl+F5 強制重整）即可看到。
- 若想看進度：GitHub → repo `galilee7989.github.io` → 上方「Actions」，最新一筆出現綠色勾勾＝完成。

## 六、在本機改東西後，如何讓網站更新（進階）

在資料夾裡開 PowerShell 或用 Claude CLI，依序：

```
git pull        （先把網頁後台的最新變更抓下來，避免衝突）
git add .
git commit -m "說明這次改了什麼"
git push        （上傳，GitHub 會自動重建網站）
```

> 重點：本機動手前**先 git pull**，因為你也會用網頁後台改東西。

## 七、換電腦或搬資料夾

網站在 GitHub 上，跟本機無關。要在新電腦或新位置繼續：

1. 安裝 Git 與 Node.js（22 版以上）。
2. 開 PowerShell，到你要放的位置，執行：
   ```
   git clone https://github.com/galilee7989/galilee7989.github.io.git galilee
   cd galilee
   npm install
   ```
3. 完成，這份就跟 GitHub 一模一樣。

> **搬資料夾請用「重新 clone」或整包剪下貼上**；不要只複製部分內容，以免弄壞 git。

## 九、常見問題

- **PDF 打不開 / 一直下載**：一般訪客沒問題。若你自己電腦裝了 IDM 等下載工具攔截，把 galilee.org.tw 加入 IDM 例外，或關掉它的瀏覽器監視。
- **改了東西網站沒變**：確認有按 Save（網頁）或有 push（本機）；等 1～2 分鐘；用 Ctrl+F5 強制重整。
- **www 開不了**：以 https://galilee.org.tw 為主即可；www 由 GitHub 自動轉址，偶有延遲。

## 十、不要做的事（避免弄壞）

- 不要手動亂改 `src/data/prayletters/` 裡的 JSON、`gallery.json` 的格式（用後台新增最安全）。
- 不要刪掉 `.github/`、`.pages.yml`、`astro.config.mjs`、`public/CNAME` 這些設定檔。
- `scripts/admin-config.json`（後台密碼）不要上傳到公開 repo（已設定自動忽略）。
